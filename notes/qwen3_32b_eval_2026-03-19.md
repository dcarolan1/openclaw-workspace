## Test Results

**Prompt:** Extract name, phone, email, vehicles, insurance policy, agent from a messy text block. Return as JSON.

**Result:** Perfect JSON. Every field correct, arrays for multi-value fields, clean formatting.

**Performance:** 129 tokens, 1.51 tok/sec (~86 seconds)

Flawless extraction. No hallucination, no missing fields, proper JSON structure. This is exactly the kind of private-data processing task where a local model shines — no data leaving the box.

## Verdict: Worth Keeping?

**Yes.** The quality jump from 8B to 32B is real and meaningful:
- Data extraction: perfect vs good
- Writing: near-final vs draft
- Reasoning: substantive vs shallow

**The tradeoff is speed.** 1.5 tok/sec means ~2-4 minutes per response. This is NOT an interactive model on this hardware. It's a **batch/background model** — use it for:
- Processing private data locally (the killer use case)
- Drafting business content that needs to be close to final
- Analysis tasks where quality matters more than speed
- Anything where you'd otherwise send sensitive data to an external API

**Keep both models.** Use qwen3:8b for speed, qwen3:32b for quality. Route accordingly.

## Recommended Local Model Stack

1. **qwen3:8b** → speed (quick rewrites, formatting, simple tasks) — ~6 tok/sec
2. **qwen3:32b** → working (private data, business writing, analysis) — ~1.5 tok/sec
3. **Claude Opus** → thinking (hard reasoning, decisions, David-facing critical output)
